<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Caddy (TLS) → localhost:WEBHOOK_PORT | Caddy termina o TLS e repassa o raw body intacto; o HMAC valida no servidor. Ponto único de exposição pública. |
| operador → setup-webhooks.js → ClickUp API | Registro autenticado com CLICKUP_TOKEN; o secret retornado é exibido uma vez para ir ao .env. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Information Disclosure | secret do webhook impresso pelo setup | mitigate | Secret exibido só na criação, via console.log efêmero, com instrução de salvar no .env (gitignored); CLICKUP_TOKEN nunca logado. |
| T-03-15 | Spoofing | endpoint público recebe POST forjado | mitigate | HMAC sempre ativo no handler (Plano 02); Caddy preserva o raw body → assinatura valida de verdade; 401 sem assinatura. |
| T-03-16 | Tampering | criação de webhooks duplicados | mitigate | setup-webhooks idempotente: find por endpoint+list_id antes de criar; PUT em vez de POST se existe (D-04). |
| T-03-17 | Information Disclosure | secrets/domínio no deploy/README.md | mitigate | Doc usa placeholders (`<seu-dominio>`), sem tokens/secret reais; .env permanece gitignored. |
| T-03-18 | Elevation of Privilege | poller reprocessa estados publicados | mitigate | ghlDedup + CF_GHL_POST_ID âncora (Plano 03); updateTask idempotente. |
| T-03-SC | Tampering | npm installs | mitigate | Nenhum pacote novo nesta fase (Caddy é binário de infra, fora do npm; smee-client descartado em D-08). Sem superfície de supply-chain npm. |
</threat_model>
