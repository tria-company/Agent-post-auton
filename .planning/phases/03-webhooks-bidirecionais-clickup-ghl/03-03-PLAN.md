<objective>
Entregar o slice vertical GHL → ClickUp via POLLING (o GHL NÃO emite webhook de post — confirmado HIGH no RESEARCH RQ1). Um módulo `src/poller/ghlStatusPoller.js` varre as tasks em `agendado` que já têm GHL Post ID, consulta `ghl.getPost(postId)` e faz o write-back: publicado → `publicado` + IG Media ID + Link publicado (D-02/SYNC-04); falha → volta para `a agendar` + Erro de publicação + LIMPA o GHL Post ID para retry (D-01/SYNC-05). Idempotência por DedupeStore `postId:status` (SYNC-06), isolamento por task (D-18).

Purpose: Fecha o laço inverso da sincronização. Sem ele, uma task agendada nunca reflete a publicação real no Instagram. Como o GHL não tem webhook, polling é o mecanismo obrigatório (SYNC-01). Os nomes REAIS dos campos vêm do smoke OQ1 do Plano 01 — este plano consome esses fatos.

Output: o módulo poller (auto-contido, exporta `pollGhlPosts`) + um arquivo de teste. O wiring do setInterval no servidor é o Plano 04 (para evitar conflito de arquivo com o Plano 02).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| poller → GHL API | Saída autenticada (token da app); a resposta do GHL é dado, mapeado por nome de campo confirmado. |
| poller → ClickUp API | Write-back autenticado; valores derivados do post GHL (IG media id/permalink) são gravados em custom fields — sem execução. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Spoofing | autenticidade do evento GHL (SYNC-02) | accept | N/A: não há webhook GHL; o polling busca a verdade diretamente na API GHL com o token da app. Documentado no módulo. |
| T-03-11 | Information Disclosure | Erro de publicação vazar URL/token em CF/comentário | mitigate | Mensagem derivada de campo de erro do post, truncada a 200 chars, sem http/pit-/pk_ (padrão pipeline D-15/T-02-03). |
| T-03-12 | Elevation of Privilege | reprocessar publicado/falha repetidamente | mitigate | DedupeStore `postId:status` (SYNC-06) + CF_GHL_POST_ID como âncora; updateTask idempotente. |
| T-03-13 | Denial of Service | uma task quebrada aborta toda a varredura | mitigate | try/catch por task (D-18); falha isolada não interrompe o loop. |
| T-03-SC | Tampering | npm installs | mitigate | Nenhum pacote novo (reusa clients e dedupe existentes). |
</threat_model>
