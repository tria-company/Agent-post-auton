<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ClickUp (internet) → Caddy → /webhook/clickup | Input não confiável atravessa aqui; só processado após HMAC válido. Caddy preserva o raw body (TLS termina no Caddy, body repassado intacto). |
| /webhook/clickup → processTask | Após HMAC + filtro, a task_id confiável é resolvida via getTask (servidor busca a verdade no ClickUp, não confia no payload para custom_fields). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | webhook forjado sem HMAC | mitigate | verifyClickUpSignature sobre o RAW body ANTES de qualquer efeito; 401 se inválido (L-05/TRIG-02). HMAC sempre ativo (sem SKIP). |
| T-03-05 | Spoofing | timing attack na comparação HMAC | mitigate | crypto.timingSafeEqual obrigatório; nunca `==` (RESEARCH "Don't Hand-Roll Security"). |
| T-03-06 | Elevation of Privilege | replay/reentrega repetida | mitigate | DedupeStore por `webhook_id:history_item_id` (TRIG-04) + guarda de CF_GHL_POST_ID no processTask. |
| T-03-07 | Information Disclosure | log da assinatura/secret | mitigate | Nunca passar `x-signature` nem CLICKUP_WEBHOOK_SECRET a log; só `step`/`dedupKey` (T-01-02); redaction pino ativa. |
| T-03-08 | Denial of Service | handler lento → timeout → reentrega storm | mitigate | Responder 200 imediato + setImmediate (Pitfall 6); processamento pesado fora do ciclo de resposta. |
| T-03-09 | Denial of Service | body gigante consome memória no concat | accept | VPS single-instance, payloads ClickUp são pequenos (<100KB); limite de tamanho fica para o endurecimento da Phase 4. |
| T-03-SC | Tampering | npm installs | mitigate | Nenhum pacote novo (node:http/node:crypto nativos; smee-client descartado em D-08). Sem superfície de supply-chain. |
</threat_model>
