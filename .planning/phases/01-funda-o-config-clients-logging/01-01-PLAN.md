| processo → `.env` | Segredos (pk_…, pit-…, locationId) entram no processo via process.env; nunca devem ir para o código, git ou logs |
| processo → API ClickUp / GHL | Saída autenticada com tokens via HTTPS; o body de erro retornado é dado externo (não confiável para log irrestrito) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Token em código/commit | mitigate | CFG-01: zero hardcode (gate grep na Task 2); `.gitignore` já cobre `.env`/`.env.*` (verificado pré-plano); `.env` nunca lido pelo código exceto via process.env |
| T-01-02 | Information Disclosure | Token/header vazado em log | mitigate | pino `redact` em `authorization`/`token` (Task 2); clients nunca logam headers de auth; mapeadores de erro logam só status+code (Task 3) |
