<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repositório git → segredos | `.env` jamais pode entrar no histórico do git; só `.env.example` (sem valores) é versionado |
| logs → operador | Linhas de log são lidas por humanos/ingest; não podem conter tokens nem headers de auth |
| API ClickUp/GHL → processo | Body de erro/resposta é dado externo; só status+code derivados vão para log/AppError |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Information Disclosure | `.env` commitado no histórico git | mitigate | Checkpoint humano bloqueante (Task 3): `git status` + auditoria de histórico de `.env`; `.gitignore` já cobre `.env`/`.env.*` |
| T-01-07 | Information Disclosure | Token impresso no boot/smoke estendido | mitigate | Checkpoint humano confirma ausência de pk_/pit_/Authorization nos logs (Task 3); redaction do logger (Plano 01-01) |
| T-01-08 | Tampering / Information Disclosure | Config inválida aceita silenciosamente | mitigate | test/config.test.js prova fail-fast com env incompleto (Task 1) |
| T-01-09 | Information Disclosure | Erro de API logado com corpo/headers sensíveis | mitigate | test/errors.test.js prova que AppError serializa só status+code (Task 1) |
| T-01-10 | Repudiation | Rotação das keys vazadas esquecida | accept | Rastreado como blocker pré-produção no STATE.md; lembrete reforçado no checkpoint (Task 3) e no README — fora do escopo bloqueante da fundação |
</threat_model>
